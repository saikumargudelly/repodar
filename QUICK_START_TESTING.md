# Quick Start: A2A Services Real-Time Testing

Get started with A2A services testing in 5 minutes.

## Step 1: Start the Backend (Terminal 1)

```bash
cd backend
source .venv/bin/activate
python -m uvicorn app.main:app --reload --port 8000
```

Wait for: `Uvicorn running on http://127.0.0.1:8000`

## Step 2: Quick Validation (Terminal 2)

```bash
cd backend
source .venv/bin/activate
python test_a2a.py
```

Expected: ✅ 9/9 tests passed

## Step 3: Full Integration Test

```bash
python test_a2a_full.py
```

Expected: ✅ 10/10 tests passed

## Step 4: Real-Time Tests

```bash
python test_a2a_realtime.py
```

Expected: ✅ 10/10 tests passed with performance metrics

## Step 5: Load Testing

```bash
python test_a2a_load.py
```

## Troubleshooting

**Backend won't start?**
```bash
# Kill existing process
pkill -f "uvicorn app.main:app"

# Start fresh
python -m uvicorn app.main:app --reload --port 8000
```

**Tests fail to connect?**
```bash
# Check if backend is running
curl http://localhost:8000/health

# If offline, start backend
python -m uvicorn app.main:app --reload --port 8000
```

**High response times (>500ms)?**
```bash
# Check system resources
top

# Reduce concurrency in load tests
# Edit test_a2a_load.py concurrency settings
```
